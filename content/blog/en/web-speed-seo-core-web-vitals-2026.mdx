## What are Core Web Vitals and why does Google use them in 2026

Core Web Vitals are three metrics Google uses to measure the real user experience on a website:

- **LCP (Largest Contentful Paint):** How long it takes for the main element of the page to appear. To be "good", it must be under 2.5 seconds.
- **INP (Interaction to Next Paint):** How quickly the website responds when the user clicks something. It must be under 200ms.
- **CLS (Cumulative Layout Shift):** Whether page elements "jump" while loading, which disorients the user. Must be under 0.1.

Google has used these metrics as a ranking factor since 2021, and in 2026 they remain a direct ranking signal. **Sites that pass all three thresholds see 24% lower bounce rates** and measurable improvements in organic traffic compared to those that don't.

## WordPress's performance problem

WordPress was designed in 2003 as a blogging system. In 2026, most WordPress sites have:

- A theme that loads CSS and JavaScript for thousands of components even if it doesn't use them
- 15-30 active plugins, each adding their own code
- Database queries for every page view
- Unoptimised images served in outdated formats

The typical result in PageSpeed Insights for a WordPress site with Elementor or Divi on mobile: **40-60 points out of 100**.

For context, Google considers a score of 90 or more "good". Below 50 is considered "poor".
